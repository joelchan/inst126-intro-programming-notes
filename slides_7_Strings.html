Properties of strings

Like lists, strings are sequences: ordered collections that support indexing, slicing, len(), and in.

But strings are not lists — and strings are immutable.

Strings are ordered

a_string = "hello world Hi"

# can be indexed
a_string[0]       # 'h'

# can be sliced
a_string[:5]      # 'hello'

# can be sorted
sorted(a_string)  # [' ', ' ', 'H', 'd', 'e', 'h', 'i', 'l', 'l', 'l', 'o', 'o', 'r', 'w']

Strings have length

a_string = "hello world Hi"
print(len(a_string))   # 14

Remember: spaces count as characters!

Strings are IMMUTABLE

a_string = "hello"
a_string.upper()     # this does NOT change a_string
print(a_string)      # still "hello"

# to preserve the change, you must reassign
a_string = a_string.upper()
print(a_string)      # now "HELLO"
Anytime you modify a string, you must assign the result to a variable to preserve the change.

Aside: string encoding

Each character has a unique numeric code.
Use ord() to see it:

print("A:", ord("A"))   # 65
print("a:", ord("a"))   # 97
print("0:", ord("0"))   # 48
print(" :", ord(" "))   # 32

'A' is 65, 'a' is 97 — completely different!

Why encoding matters

s1 = "James"
s2 = "james"
s3 = "James "
print(s1 == s2)   # False — different case
print(s1 == s3)   # False — trailing space
Always normalize your strings before comparing them!

Cleaning / normalizing strings

  • .lower() / .upper() — normalize case
  • .replace() — replace parts of a string
  • .strip() — remove leading/trailing whitespace

Chaining methods

def normalize_sales_record(sale):
    return sale.replace("$", "").replace(",", "")

sales_record = "$1,000,000"
cleaned = normalize_sales_record(sales_record)
print(cleaned)   # "1000000"
Chaining works because each .method() returns a new string, which you can immediately call another method on.

Normalizing for comparison

def normalize_string(s):
    return s.upper().strip()

n = " Josh Lyman"
m = "JOSH LYMAN"
print(n == m)   # False

n_normal = normalize_string(n)
m_normal = normalize_string(m)
print(n_normal == m_normal)   # True!

Parsing strings with .split()

Use a separator to divide a string into a list of parts.

  • Parse an email
  • Parse a URL
  • Parse a sentence into words
  • Parse a timestamp

Example: parse an email

email = "joelchan@umd.edu"

elements = email.split("@")
print(elements)   # ['joelchan', 'umd.edu']

username = elements[0]
print(username)   # 'joelchan'

Multi-step splitting

email = "joelchan@umd.edu"

split1 = email.split("@")       # ['joelchan', 'umd.edu']
domainserver = split1[1]         # 'umd.edu'

split2 = domainserver.split(".") # ['umd', 'edu']
domain = split2[1]               # 'edu'
print(domain)

REMEMBER:
Strings are immutable!

The change is lost!

a = "Hello"
b = "WORLD"
a.lower()   # this returns "hello" but...
b.lower()   # this returns "world" but...
print(a, b) # still "Hello WORLD"!
Fix: reassign the result! 
a = a.lower()   # now a is "hello"
b = b.lower()   # now b is "world"

String formatting with f-strings

Composing strings from variables

The basics

Prefix with f, then put variables in {} slots:

msg = "hello"
friend = "rony"
name = "anna"
output = f"{msg} {friend}, my name is {name}!"
print(output)
hello rony, my name is anna!

Expressions inside {}

weight_kgs = 120
print(f"{weight_kgs} is {weight_kgs * 2.2} lbs")
# 120 is 264.0 lbs
birth_year = 1956
this_year = 2023
name = "Joel"
msg = f"Happy birthday, {name}! You are {this_year - birth_year} this year!"
print(msg)
# Happy birthday, Joel! You are 67 this year!

f-strings in loops

names = ["Joel", "Sarah", "Michael", "Kacie"]
for name in names:
    message = f"Welcome, {name}!"
    print(message)
Welcome, Joel!
Welcome, Sarah!
Welcome, Michael!
Welcome, Kacie!

f-strings for debugging

sales = ["$100", "$250", "$500"]

for i in range(len(sales)):
